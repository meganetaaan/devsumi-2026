# Presentation Review Rubric

## 1. Story Clarity
- 導入で課題が提示されているか。
- 中盤の説明が結論の根拠になっているか。
- まとめで持ち帰りが明確か。

## 2. Slide to Note Consistency
- `index.html` のスライド順とスライド内ノートの対応が一致しているか。
- ノートの説明対象が実在スライドと一致しているか。
- スライドにない主張をノートで断定していないか。
- `Note:` または `aside.notes` が Reveal.js として有効な位置にあるか。

## 3. Timing and Pacing
- 総尺に対して各セクションが現実的か。
- デモ、質疑、まとめの時間を食い合っていないか。
- 長文化しやすい区間に削減案があるか。

## 4. Evidence and Credibility
- 数値、年次、固有名詞が既存資料と矛盾していないか。
- 根拠が曖昧な断定表現をしていないか。

## 5. Delivery Risk
- デモ依存箇所にフォールバック説明があるか。
- 早口になりやすい文が集中していないか。
- 聴衆が置いていかれる専門用語の連発がないか。

## Severity Guide
- `Critical`: 本番で失敗する可能性が高い問題。
- `Major`: 伝達効果を大きく損なう問題。
- `Minor`: 品質を上げるための改善提案。
