# Slide Checklist

## Story
- タイトルで結論が想像できるか。
- 導入で課題、本文で方法、終盤で成果、結論で持ち帰りが揃っているか。
- セクション間のつながりが口頭で自然に説明できるか。

## Density
- 1スライド1メッセージになっているか。
- 1スライドの箇条書きが多すぎないか。
- 口頭補足前提の情報を本文に詰め込みすぎていないか。

## Visual
- 既存の CSS クラスとレイアウトを再利用できているか。
- 画像パス、`alt`、リンク先が破綻していないか。
- 読みやすさを崩す過度な装飾を入れていないか。

## Fact
- 数値と固有名詞が `IDEA.md`, `reference/`, `index.html` ノートと矛盾していないか。
- 根拠が不明な断定表現をしていないか。

## Notes Sync
- スライド本文の変更が `Note:` ノート内容と矛盾していないか。
- スライド順変更時にノートの対応がずれていないか。

## Runtime
- `npm run build` が成功するか。
